+[mulle-objc about]

mulle-objc is a new way to run Objective-C code on various platforms. It consists of the following parts:

Compilermulle-clang
Debuggermulle-lldb
Runtimemulle-objc
Class libraryMulleFoundation
C librarymulle-c
ToolsMulleSDE

Its components are licensed under BSD terms, which is an Open Source license.

See "About mulle-objc" for more in-depth information and technical background.

+[mulle-objc goals]

  • Improve and maintain Objective-C for use in the next decades
  • Run everywhere C runs
  • Everything faster than everyone else
  • Enable static linking
  • Enable massive threading
  • The runtime should be completely unloadable
  • Should work in real time applications
  • No magic
  • No locking
  • Keep the Spirit of C

+[mulle-objc politics]

mulle-objc runs on Linux, FreeBSD, MacOS and Windows. There is nothing that should prevent it from running on Android or any other OS, that's 32 bit or more. As long as there is C and clang compiles for it, Objective-C works too. If your code is compatible with mulle-objc it has gained, by the mere existence of this project, portability and longevity and therefore worth.

It would be nice to lobby mulle-objc into debian.

The compiler is a fork of clang. One of the main goals of the community is to integrate the compiler changes back into the mainline of clang. For that "Evidence of a significant user community" is one of the requirements.
